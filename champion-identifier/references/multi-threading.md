# Multi-Threading and Account Mapping

## Recommended Sequence

1. Start With: Champion #1 (highest probability)
2. Parallel Outreach: Champion #2 (different department, covers more ground)
3. Economic Buyer Access: ask Champion #1 to introduce the decision maker
4. Technical Validator: connect with the relevant technical evaluator
5. User Buy-In: gather feedback from an end-user representative

## Coverage Map

```
Economic Buyer (Decision): [Name, Title]
        ^
Champion (Advocate): [Champion #1]
        ^
Technical (Validate): [Technical Person]
        ^
Users (Adopt): [User Team]
```

## Why Multi-Thread

- Single-threaded deals die if the contact leaves or loses interest
- Different perspectives reveal the full picture
- Multiple internal conversations build momentum
- Reduces the risk of being ghosted
